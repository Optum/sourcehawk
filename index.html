Sourcehawk is an extensible compliance as code tool which allows development teams to automate custom compliance scans on their source code.

Installation

Sourcehawk can be installed using the installation scripts from GitHub.

Linux

curl https://raw.githubusercontent.com/Optum/sourcehawk/main/install-linux.sh | bash

Debian

You’ll need to be able to sudo for this one.

curl https://raw.githubusercontent.com/Optum/sourcehawk/main/install-debian.sh | bash

Mac

curl https://raw.githubusercontent.com/Optum/sourcehawk/main/install-mac.sh | bash

Usage Manuals

Sourcehawk is available to be run as a command line tool. The following commands are available.

sourcehawk(1)

Name

sourcehawk - Watch over your source like a hawk…​

Synopsis

sourcehawk [-hV] [COMMAND]

Description

Watch over your source like a hawk…​

Options

-h, --help

Show this help message and exit.

-V, --version

Print version information and exit.

Commands

help

Displays help information about the specified command

scan, flyover, survey

Runs a scan on the source code

validate-config, vc

Validate configuration

fix, correct, resolve

Fix source based on configuration

validate-config(1)

Name

validate-config - Validate configuration

Synopsis

validate-config [-hV] <configFilePath>

Description

Validate configuration

Options

-h, --help

Show this help message and exit.

-V, --version

Print version information and exit.

Arguments

<configFilePath>

The path to the configuration file, use '-' to supply configuration from stdin

Default: sourcehawk.yml

scan(1)

Name

scan - Runs a scan on the source code

Synopsis

scan [-hV] [-fow] [-f=<outputFormat>] [-v=<verbosity>] [[-c=config-file-path] | -cfu=config-file-url] [REPO-ROOT] [COMMAND]

Description

Runs a scan on the source code

Options

-c, -cf, --config-file=config-file-path

The configuration file, can be relative to REPO-ROOT, or absolute

Default: sourcehawk.yml
-cfu, --config-file-url=config-file-url

The configuration file URL

-f, --output-format=<outputFormat>

Output Format, valid values: CONSOLE, TEXT, JSON, MARKDOWN

Default: CONSOLE
-fow, --fail-on-warnings

Whether

Default: false
-h, --help

Show this help message and exit.

-v, --verbosity=<verbosity>

Verbosity of output, valid values: ZERO, LOW, MEDIUM, HIGH

Default: HIGH
-V, --version

Print version information and exit.

Arguments

[REPO-ROOT]

The repository root on the file system to scan relative to, defaults to current directory

Commands

help

Displays help information about the specified command

fix(1)

Name

fix - Fix source based on configuration

Synopsis

fix [-dhV] [-f=<outputFormat>] [-v=<verbosity>] [[-c=config-file-path] | -cfu=config-file-url] [REPO-ROOT] [COMMAND]

Description

Fix source based on configuration

Options

-c, -cf, --config-file=config-file-path

The configuration file, can be relative to REPO-ROOT, or absolute

Default: sourcehawk.yml
-cfu, --config-file-url=config-file-url

The configuration file URL

-d, --dry-run

Display fixes which would be performed, but do not perform them

-f, --output-format=<outputFormat>

Output Format, valid values: CONSOLE, TEXT, JSON, MARKDOWN

Default: CONSOLE
-h, --help

Show this help message and exit.

-v, --verbosity=<verbosity>

Verbosity of output, valid values: ZERO, LOW, MEDIUM, HIGH

Default: HIGH
-V, --version

Print version information and exit.

Arguments

[REPO-ROOT]

The repository root on the file system to scan relative to, defaults to current directory

Commands

help

Displays help information about the specified command
